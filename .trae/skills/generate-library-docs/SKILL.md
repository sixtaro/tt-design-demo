# Skill: Component Library Documentation Generator

## 1. 执行触发点
- 用户要求生成组件库综合文档
- 用户要求生成多个组件的文档
- 用户要求整理组件库文档

## 2. 自动化执行步骤

### 2.1 源码嗅探 (Inspection)
- 遍历 `src/components/` 目录，获取所有组件列表
- 读取每个组件的 `index.js`，分析其功能定位
- 提取 `propTypes` 中的所有属性名、类型及默认值
- 从 `src/utils/version-config.js` 获取组件版本号
- 读取 `*.stories.jsx` 获取使用示例

### 2.2 文档结构生成
- **文档标题**: 组件库名称 + 技术文档
- **技术栈说明**: 明确标注基于 React 17.0.1 和 Ant Design 4.24.8
- **组件分类**: 按通用、数据录入、数据展示等分类组织
- **每个组件包含**:
  - 组件名称
  - 组件说明
  - 版本信息
  - 基础用法
  - API 表格（属性、说明、类型、默认值）
- **文档规范**:
  - 文件名使用英文
  - 内容使用中文
  - 代码块不转义
  - 保持正确的 Markdown 格式

### 2.3 文件写入
- 在 `docs/` 目录下创建综合文档
- 默认文件名：`tt-design-components.md`

## 3. 编写标准 (Documentation Standards)
- **分类组织**: 组件按功能分类（通用、数据录入、数据展示等）
- **API 表格**: 必须包含"属性"、"说明"、"类型"、"默认值"四列
- **代码格式**: 代码块中的 JSX 标签不转义
- **版本信息**: 每个组件标注版本号和库版本号
- **技术栈声明**: 明确标注基于 React 17.0.1 和 Ant Design 4.24.8 构建

## 4. 交互逻辑
- 文档生成后，列出所有包含的组件
- 询问用户是否需要添加更多组件
- 询问用户是否需要润色文档内容
